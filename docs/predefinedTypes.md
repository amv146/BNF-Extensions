# Predefined Types

When creating the default grammar for a project based on a grammar file, the following strings are mapped to the corresponding types based on other programming languages:

## Constants
- `false`
- `null`
- `true`
- `undefined`

## Keywords
- `abstract`
- `as`
- `assert`
- `async`
- `await`
- `break`
- `case`
- `catch`
- `class`
- `const`
- `constructor`
- `continue`
- `declare`
- `default`
- `delete`
- `do`
- `else`
- `enum`
- `export`
- `extends`
- `finally`
- `for`
- `from`
- `function`
- `if`
- `implements`
- `import`
- `in`
- `instanceof`
- `interface`
- `is`
- `let`
- `module`
- `namespace`
- `new`
- `of`
- `package`
- `private`
- `public`
- `require`
- `return`
- `static`
- `super`
- `switch`
- `then`
- `this`
- `throw`
- `try`
- `type`
- `typeof`
- `var`
- `while`
- `with`
- `yield`

## Types
- `any`
- `boolean`
- `byte`
- `char`
- `double`
- `float`
- `int`
- `long`
- `number`
- `object`
- `short`
- `string`

## Functions
Any string that is not a constant, keyword, or type is mapped to a (built-in) function.
